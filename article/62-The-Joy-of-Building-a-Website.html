建站的乐趣

2026.06.12 · 建站记录

这些年见过太多网站诞生，也见过太多网站消失。而这个网站却意外地活了下来。十二年前，我把它放在 Github Pages 上，用最简单的方式搭建起来。如今再回头看，或许简单本身就是一种生命力。

这个网站建在 Github 上，是在我年轻的时候建立的，嗯，让我翻一下仓库提交记录，找到了，更准确地说，是12年前，我建立了这个网站。

Github的名声在外是以代码资料托管仓库而闻名的，后来有一天，它开启了免费 Pages 服务。当时我突发奇想，我何不在 Github 上建个网站呢。

在那时，也就是十几年前，或者更早些时候，个人博客方兴未艾。亲爱的，那时还不流行什么公众号、抖音、小红书，如果你没有亲身经历过 Blog 、网站联盟、友情链接、社区论坛 时代，说明你只是新一代网民。作为当时的“资深网民”，有一个自己的博客网站几乎是标配。从网易博客、新浪博客等平台开始，到可以自定义博客网站的 Wordpress 系统兴起之后，更进一步的还会购买独立域名，再租一个 VPS 空间，网站基本就可以完全自主化了。就象人类寻求自由文明的典型发展史，如果没有自由不能完全控制自己的网站，总觉得象有什么把柄被捏在别人手里。

有了完全自主权还没完，建站的终极形态仍然继续往前走。大家知道，Wordpress 是使用 PHP + Mysql 数据库写成的动态网站，网站的评论系统、分类系统、Tag系统、相关文章、动态特效等设置越来越复杂，甚至有大佬完全自己写博客发布系统。

所有这些我都经历过，但，我们到底最后写了并发布了多少文章？我们的网站能存活多少年？最常见的是今天推出网站，一年后或两年后消失，能撑过三年的很少。

也许是累了，当 Github 推出免费 Pages 服务时，我不禁想，我何不在这里建个网站呢？

也许是老了，动态网站折腾了很多年，当我想静下心来的时候，突然想不如做一个全静态网站，回归网络的本源，只用 html+css 就行，任何浏览器都兼容，我相信 Github 不会倒，只要写一个网站仓库提交即可，不管多久比如5年、10年后，网站也依然能访问。我固执地认为只有能通过时间考验的网站才是有意义的。

我已经过了有时间和精力去折腾的年代。不如把重心放在生活工作的其它方面，这里只要有一个角落即可，作为一个出口也好，作为一个时代的纪念也罢。
