## AI建站-静态网站的春天

AI 是近一两年里最热络的话题之一。有说 AI智能 很快会代替人类工作导致大规模失业的，有说 AI智能 元年已至，即将超越人类，甚至毁灭人类的趋势已是不可阻挡的。对于AI，不管是作恶说，还是本善论，我作为人类之中最普通的一员，就象思考人性本源之善恶谜题，沸沸扬扬中却总不得要领。

但AI智能的进步是有目共睹的。我怀疑当前在工作生活中从未使用过AI的人应该是少数了吧，有时碰到难题或疑问，与其去搜索引擎找答案，不如打开AI应用在聊天框里直接问，那样似乎更方便快捷，而且出来的结果也更整齐有序。

直到那天早晨之前，我使用 AI 的主要方式就是如此，就算当 Codex 横空出世，于我似乎也影响不大。虽然很早些时候曾学过 Delphi ，写过SQL数据库应用，后来还学了点 Python 和 Golang 语言皮毛（本来想学C或C++语言，但不知在哪儿听说Go语言更牛逼），建站时又学了点 html（Dreamweave）、 CSS ，Vim 编辑器（写代码）等。

所有这些，在技术上我只做到够用即可。研究得并不深入，说是皮毛其实并不是谦虚之词。而且这些片断的技术历史，在时间的长河里已遗忘殆尽，真的，当很长时间不再使用某种技能时，这个技能就逐渐弱化了。

但有一点是永远不会遗忘的。那就是编程的思路或者思维方式。比如我明白如何去读取一组数据，把数据存储在数组中，利用循环语句处理，我也明白数据库结构中的多个数据表是如何通过主键字段关联起来生成查询视图的。

有一次我在书店里看到有本书的名字叫“数学之美”，啊，我感同深受。有时候一段优雅的程序代码，就能让我们领略到那种美。

好了，把时间拉回到现在。除了使用固定的 html 维护一下网站外，我已经很多年不碰其它代码了。主要是没了需求，工作逐渐向管理岗位转变，生活也崇尚极简主义懒得折腾了。

但那天早晨，我突然想到，我不是还有一个网站需要维护嘛，每次提交新文章时，都要按模板手工创建几个网页才能完成任务，那么我何不让 Codex 来自动完成呢？可以把整体发布流程封装成 Skill 技能，以后只要把文章内容发给 AI ，说一句请为我发布文章即可。对，这样的发布才显得更优雅。

我抱着试一试的态度，把这个想法告诉了 [ChatGPT](https://chatgpt.com)，它马上回应了我，是的，完全可以实现。

然后我带着兴奋的心情去了趟卫生间。坐在马桶上把我的想法和要求发给 [ChatGPT](https://chatgpt.com)，结果出乎我意料的快，在我走出卫生间之前，它已经把做好的网页设计稿打包发我下载了。

结果还行，我觉得至少比我自己做得好看。
